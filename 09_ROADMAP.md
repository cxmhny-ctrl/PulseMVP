# Pulse Roadmap

## Phase 0: Validation Prototype

Goal:

```text
Validate that users understand and want the stuck-to-action loop.
```

Build:

- landing page
- clickable product mock
- widget demo
- SMS demo using Twilio
- simple AI task breakdown prototype
- manual Wizard-of-Oz detection
- beta signup form

Validate:

- Does the concept resonate?
- Do ADHD users trust the product?
- Which channel feels most useful?
- Is one-time pricing attractive?
- Do users prefer this over reminders/ChatGPT?

## Phase 1: MVP

Goal:

```text
Create repeated stuck-to-action transitions.
```

Build:

- onboarding
- voice task capture
- simple task model
- local notifications
- widget surface
- SMS reply/deep-link loop
- Stuck Mode AI
- local fallback templates
- weekly summary
- sensitivity controls
- WAI analytics

Success:

- active users have repeated weekly interventions
- users report Pulse helped them start
- low false-positive annoyance
- SMS/widget prove meaningful

## Phase 2: Personalization

Goal:

```text
Make Pulse better for each user over time.
```

Build:

- best channel detection
- tone preference learning
- sensitivity learning
- friction pattern recognition
- time-of-day effectiveness
- easier step generation based on past behavior

## Phase 3: Family Profiles

Goal:

```text
Support ADHD households without creating shame or conflict.
```

Build:

- linked profiles
- parent/child permissions
- shared routines
- encouragement prompts
- weekly family summaries
- privacy controls for children/teens

## Phase 4: Professional Dashboard

Goal:

```text
Let ADHD coaches and therapists recommend Pulse as a client-support tool.
```

Build:

- therapist accounts
- client invites
- consent flow
- weekly client summary
- friction patterns
- channel effectiveness
- setting recommendations
- exportable report

## Phase 5: Advanced Detection

Goal:

```text
Improve stuck detection while preserving trust.
```

Build:

- optional behavioral detection
- repeated unlock detection where platform allows
- app switching loop detection where allowed
- calendar/task mismatch signals
- Android foreground active detection mode
- iOS Shortcuts/Focus integrations
- personalized stuck probability model

## Phase 6: Ecosystem Integrations

Goal:

```text
Let Pulse fit into existing workflows without becoming a task manager.
```

Possible integrations:

- Google Calendar
- Apple Calendar
- Todoist
- Notion
- reminders
- smart speakers
- ADHD coaching platforms
- therapist EHR export only if legally appropriate

## Strategic Sequencing Rule
Never build complex productivity features before proving the stuck-to-action loop.

Avoid becoming:

- a to-do app
- a habit tracker
- a calendar app
- a project manager
- a therapy platform

Pulse should stay focused on task initiation.
